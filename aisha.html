GitHub

منصة استضافة وتعاون للمشاريع البرمجية

فوائد ميزات ابدأ الآن سجّل دخول

إدارة، مشاركة وتطوير الكود مع GitHub

GitHub هي منصة تعتمد على نظام التحكم بالإصدارات Git وتوفر أدوات للتعاون بين المطورين، إدارة المشاريع، واستضافة الشفرات المصدرية. مناسبة للفرق من صغيرة إلى كبيرة وللمشاريع المفتوحة والمغلقة.

ابدأ مجانًا اطّلع على الميزات

نموذج سريع: إنشاء مستودع ومزامنته

# على جهازك
git init
git add .
git commit -m "النسخة الأولية"
git remote add origin https://github.com/username/repo.git
git push -u origin main

هذا المثال يشرح كيفية رفع مشروع محلي إلى مستودع GitHub.

فوائد استخدام GitHub

التتبع والتحكم بالإصدارات

القدرة على الرجوع إلى نسخ سابقة من المشروع، مقارنة التغييرات، وفهم تاريخ التطوير بسهولة.

التعاون والعمل الجماعي

فروع (branches)، طلبات السحب (Pull Requests)، ومراجعات الشفرة تجعل العمل المشترك منظمًا وآمنًا.

التكامل والأتمتة

GitHub Actions يوفّر إمكانيات CI/CD لتشغيل اختبارات، بناء التطبيقات، ونشرها تلقائيًا.

ميزات مهمة

  • مستودعات عامة وخاصة: خيار للمشاريع المفتوحة أو الاحتفاظ بالكود خاصًا.
  • Issues و Projects: أدوات لإدارة المهام، تتبع الأخطاء، وتنظيم سير العمل.
  • Actions: أتمتة سير العمل (اختبارات، بناء، نشر).
  • Pages: استضافة مواقع ثابتة من مستودعات GitHub بسهولة.
  • Codespaces: بيئة تطوير سحابية جاهزة للعمل.
  • حماية الفروع: سياسات للمراجعات وإجراءات الأمان قبل الدمج.

ابدأ استخدام GitHub — خطوات سريعة

  1. سجّل حسابًا على github.com.
  2. أنشئ مستودعًا جديدًا (Repository) — اختر اسمًا ووصفًا واضحًا.
  3. اربط المستودع بمشروعك المحلي باستخدام git وادفع التغييرات (push).
  4. استخدم فروعًا منفصلة للعمل على ميزات جديدة وفتح Pull Requests للمراجعة.
  5. قم بإعداد Actions للاختبارات وعمليات النشر التلقائي.

هل تريد مساعدة في إنشاء مستودع أو إعداد Actions؟

أستطيع توليد ملفات إعداد (مثل .github/workflows) أو كتابة ملف README احترافي بالعربية.
